Analyse des Algorithmes de Tri

QuickSort MergeSort BubbleSort Java

Description

Analyse expérimentale de l'impact du désordre sur les performances des algorithmes de tri.

Ce projet universitaire réalisé dans le cadre de la Licence Informatique à l'Université de Caen Normandie vise à comprendre comment la quantité et la répartition du désordre influencent l'efficacité des algorithmes de tri en termes de comparaisons, d'échanges et de temps d'exécution.

1. Générateurs de données :

  • Distribution uniforme : Toutes les valeurs ont la même probabilité d'apparaître
  • Distribution normale : Valeurs centrées autour d'une moyenne avec un écart-type
  • Désordre aléatoire : Échanges aléatoires entre positions quelconques
  • Désordre local : Échanges uniquement entre éléments voisins

2. Générateur de séries :

  • Schéma répétitif : Blocs de valeurs identiques (ex: 1 1 1 2 2 2)
  • Schéma séquentiel : Répétition cyclique d'une séquence (ex: 1 2 3 1 2 3)
  • Schéma mélangé : Répartition aléatoire des valeurs
  • Calcul d'entropie : Mesure du niveau de désorganisation des données

Objectifs

  • Comparer les performances des algorithmes de tri sur différents types de données
  • Mesurer l'impact du désordre initial sur le temps d'exécution
  • Analyser le nombre de comparaisons et d'échanges effectués
  • Étudier la relation entre entropie et complexité algorithmique

Algorithmes Implémentés

QuickSort

Algorithme diviser-pour-régner avec complexité O(n log n) en moyenne

Partitionnement Récursivité

MergeSort

Tri par fusion avec complexité O(n log n) garantie

Diviser-pour-régner Stable

BubbleSort

Tri à bulles avec complexité O(n²)

Simple Comparaisons adjacentes

InsertionSort

Tri par insertion avec complexité O(n²)

Efficient sur petits jeux Partiellement triés

Méthodologie

1. Conception modulaire

Architecture en packages Java (generator, distribution, désordre, sorting) avec diagramme UML

2. Génération des données

Création de jeux de tests variés selon différentes distributions et niveaux de désordre

3. Expérimentations

Mesure des performances (temps, comparaisons, échanges) pour chaque algorithme

4. Analyse des résultats

Comparaison des algorithmes et visualisation via des graphiques Python/Matplotlib

Résultats Clés

  • Distribution uniforme : QuickSort et MergeSort montrent les meilleures performances
  • Distribution normale : InsertionSort plus performant qu'en distribution uniforme
  • Données séquentielles : Algorithmes adaptatifs profitent de la structure préexistante
  • Données mélangées : QuickSort et MergeSort restent robustes face au chaos

Technologies Utilisées

Langage principal

Java 11

Visualisation

Python, Matplotlib

Automatisation

Scripts Bash, JSON

Défis & Perspectives

Difficultés rencontrées

  • Génération de séries selon une entropie cible
  • Optimisation des mesures de performance
  • Gestion des très grands jeux de données

Améliorations possibles

  • Intégration d'algorithmes hybrides
  • Analyse de la consommation mémoire
  • Interface graphique plus interactive

Conclusion

Ce projet a permis une analyse approfondie des algorithmes de tri et de leur comportement face à différents types de données. L'approche modulaire et l'automatisation des tests ont facilité les comparaisons objectives. Les résultats confirment l'importance de choisir l'algorithme adapté à la nature des données à trier, et ouvrent des perspectives intéressantes pour l'optimisation des tris hybrides.
