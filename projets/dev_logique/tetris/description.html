Jeu d'Assemblage de Formes

Java MVC Design Patterns POO

Description

Implémentation d'un jeu d'assemblage de formes avec architecture MVC et design patterns.

Ce projet universitaire réalisé dans le cadre du cours "Méthode de Conception" à l'Université de Caen Normandie consiste en la création d'un jeu où le joueur doit assembler différentes formes pour occuper le moins d'espace possible, avec une architecture modulaire et l'utilisation de plusieurs design patterns.

Principe du jeu : Inspiré du Tetris, le joueur doit assembler des pièces en les faisant pivoter et déplacer pour minimiser l'espace occupé. Le score est calculé en fonction du plus petit rectangle englobant toutes les pièces.

Applications du Projet

JeuxAssemblageMVC

Application principale implémentant le jeu avec interface graphique et ligne de commande, utilisant la librairie PiecePuzzle.

Structure MVC complète avec observers

PiecePuzzle

Librairie contenant les composants de base du jeu : pièces, plateau, et mécanismes fondamentaux.

Génération aléatoire des pièces

Architecture MVC

Organisation Modèle-Vue-Contrôleur

Le projet utilise le pattern Observer pour implémenter l'architecture MVC :

  • Modèle (Plateau) : Gère la logique métier du jeu
  • Vue (BoardView) : Affiche l'état du plateau graphiquement
  • Contrôleur : Gère les interactions utilisateur

La classe AbstractListenableModel agit comme observable, tandis que BoardView implémente ListeningModel comme observateur.

Design Patterns Implémentés

Pattern Strategy

Utilisé pour les différentes stratégies de placement des pièces :

  • ArbitraryStrategy : Configuration prédéfinie avec coordonnées fixes
  • RandomStrategy : Placement aléatoire des pièces
  • Interface PlacementStrategy commune

Permet de changer facilement de stratégie à l'exécution.

Pattern Factory

La classe PieceFactory centralise la création des pièces :

  • Encapsule les détails de construction
  • Permet création aléatoire ou configurée
  • Méthodes fluides pour personnalisation
  • Facilite l'ajout de nouveaux types de pièces

Chain of Responsibility

Gère la validation du placement des pièces :

  • AbstractValidator : Classe de base pour la chaîne
  • OverflowValidator : Vérifie les débordements
  • OverlappingValidator : Détecte les chevauchements
  • Méthode buildChain pour construction flexible

Permet d'ajouter facilement de nouveaux validateurs.

Fonctionnalités du Jeu

Mode Joueur

  • Déplacement des pièces dans l'aire de jeu
  • Rotation des pièces pour optimisation
  • Calcul du score basé sur l'espace occupé
  • Limite d'actions configurables

Mode Automatique

  • Joueur robot avec différentes stratégies
  • Configuration via pattern Strategy
  • Comparaison des performances
  • Extensible à de nouvelles stratégies

Gestion des Pièces

  • Génération aléatoire ou configurée
  • Factory Method pour création flexible
  • Divers types de formes disponibles
  • Système de validation robuste

Interfaces

  • Interface graphique complète
  • Version ligne de commande
  • Même modèle pour les deux interfaces
  • Observers pour mise à jour automatique

Conclusion

Ce projet a été une opportunité précieuse pour approfondir notre compréhension des concepts de conception logicielle, notamment l'architecture MVC et les design patterns. La mise en pratique de ces principes sur un jeu concret nous a permis de consolider nos compétences en programmation orientée objet et en résolution de problèmes complexes.
