CoreWar - Simulation de Combat de Programmes

Java Algorithme Génétique RedCode Parallélisme

Description

Implémentation d'un simulateur CoreWar avec algorithme génétique pour générer des programmes optimisés.

Ce projet universitaire réalisé dans le cadre de ma licence d'informatique à l'Université de Caen Normandie consiste en la création d'un environnement CoreWar complet, où plusieurs programmes écrits en RedCode (un langage proche de l'assembleur) s'affrontent dans une mémoire virtuelle partagée.

CoreWar est un jeu de programmation où plusieurs programmes s'affrontent au sein d'une mémoire virtuelle. Le but est d'être le dernier programme à avoir au moins un processus en vie.

Fonctionnalités Implémentées

Moteur CoreWar

  • Mémoire virtuelle circulaire
  • Support de la norme ICWS-88
  • 11 instructions RedCode implémentées
  • 5 modes d'adressage supportés
  • Gestion des processus (SPL, DAT)

Algorithme Génétique

  • Génération de programmes optimisés
  • Méthodes de sélection (élitisme, roue)
  • Mécanismes de mutation et fusion
  • Benchmark des programmes générés

Parallélisme

  • Thread pool pour l'évaluation
  • Synchronisation avec wait/notify
  • Optimisation des temps de calcul
  • Gestion des ressources

Interface Graphique

  • Visualisation de la mémoire
  • Affichage des programmes en couleurs
  • Têtes de lecture visibles
  • Menu de sélection des programmes

Détails Techniques

Le simulateur CoreWar est implémenté en Java avec une architecture modulaire :

  • Memory : Gère la mémoire virtuelle circulaire
  • Parser : Transforme les fichiers RedCode en instructions exécutables
  • Interpreter : Exécute les programmes tour par tour
  • Cell : Représente une case mémoire avec instruction et opérandes

Le parser effectue deux passes pour gérer les labels et constantes, et vérifie la validité des instructions selon ICWS-88.

L'algorithme génétique implémente :

  1. Sélection : Élitisme ou roue stochastique
  2. Mutation : Modification aléatoire d'instructions
  3. Fusion : Combinaison de deux programmes
  4. Évaluation : Duels entre programmes avec scoring

Le benchmark évalue les programmes sur 6 métriques clés comme le temps de survie, la fréquence d'écriture, etc.

Implémentation d'une thread pool pour accélérer l'évaluation :

  • Classe Pool gère la répartition des tâches
  • Threads Duel évaluent les programmes en parallèle
  • Synchronisation avec wait/notify
  • Gestion propre de la fin d'exécution

Résultats Obtenus

Succès

  • Moteur CoreWar pleinement fonctionnel selon ICWS-88
  • Génération de programmes compétitifs en 10 générations
  • Interface graphique intuitive pour visualiser les combats
  • Accélération significative grâce au parallélisme
  • Programmes générés capables de battre des programmes manuels

Défis

  • Problème de consanguinité dans les premières versions
  • Génération aléatoire initiale peu efficace
  • Synchronisation délicate des threads
  • Optimisation des critères d'évaluation

Évolution des métriques

Le benchmark montre une amélioration significative sur les métriques clés après optimisation de l'algorithme génétique :

  • Fréquence d'écriture augmentée
  • Pourcentage d'occupation mémoire optimisé
  • Nombre de processus créés et maintenus en vie
  • Couverture mémoire améliorée

Conclusion et Perspectives

Ce projet complet m'a permis de maîtriser plusieurs concepts avancés en programmation : l'implémentation d'un langage spécifique (RedCode), les algorithmes génétiques, la programmation concurrente et la création d'interfaces graphiques. La résolution des problèmes de synchronisation et l'optimisation de l'algorithme génétique ont particulièrement renforcé mes compétences en résolution de problèmes complexes.
