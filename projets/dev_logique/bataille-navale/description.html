Bataille Navale - Java MVC

Java Architecture MVC Algorithmes Interface Graphique

Description

Implémentation du jeu classique de Bataille Navale avec une architecture MVC robuste.

Ce projet universitaire réalisé à l'Université de Caen Normandie met en œuvre une conception logicielle solide suivant le pattern Modèle-Vue-Contrôleur, avec une attention particulière portée sur l'optimisation algorithmique et la qualité du code.

La Bataille Navale est un jeu stratégique où deux joueurs s'affrontent en tentant de couler la flotte adverse. Ce projet inclut une interface graphique et une version en ligne de commande, avec une IA comme adversaire.

Fonctionnalités Implémentées

Architecture MVC

  • Modèle totalement indépendant de la vue
  • Double interface (graphique et CLI)
  • Communication via observateurs
  • Packages bien organisés

Intelligence Artificielle

  • Joueur IA avec stratégie aléatoire
  • Optimisation des tirs (pas de répétition)
  • Complexité algorithmique réduite
  • Détection des bateaux coulés

Optimisations

  • Structure de données efficace
  • Accès direct aux cases (O(1))
  • Gestion mémoire optimisée
  • Tests unitaires

Interface Graphique

  • Grilles interactives
  • Feedback visuel (tirs touchés/ratés)
  • Affichage des bateaux coulés
  • Design personnalisé

Détails Techniques

Structure modulaire respectant strictement le pattern MVC :

  • Modèle : Gestion des grilles, flottes, règles du jeu (totalement indépendant de l'UI)
  • Vue : Interface graphique (Swing/JavaFX) et interface texte (CLI)
  • Contrôleur : Médiateur entre modèle et vues, gestion des événements

Méthodes clés : GameModel, GameView, GameController

Optimisations algorithmiques majeures :

  1. Détection des impacts : Utilisation d'un tableau 2D pour accès direct (O(1))
  2. Gestion des tirs IA : Mémorisation des cases déjà testées
  3. Vérification des bateaux coulés : Algorithme optimisé en O(n)
  4. Placement des bateaux : Validation des positions sans chevauchement

Pratiques de développement exemplaires :

  • Documentation Javadoc complète
  • Tests unitaires pour les fonctions critiques
  • Gestion d'erreurs robuste
  • Code découpé en méthodes courtes et spécialisées
  • Nommage clair et cohérent
