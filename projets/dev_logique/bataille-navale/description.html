Bataille Navale - Java MVC

Java Architecture MVC Algorithmes Interface Graphique

Description

Implémentation complète du jeu de Bataille Navale avec architecture MVC et interface graphique Swing.

Ce projet Java implémente une version complète de la bataille navale avec deux modes de jeu : humain contre IA et IA contre IA. Le système suit une architecture MVC stricte avec une séparation claire des responsabilités.

Points clés : Le projet inclut une interface graphique interactive avec des grilles cliquables, un système de tour par tour, une IA basique avec stratégie aléatoire, et une gestion complète des règles du jeu.

Fonctionnalités Implémentées

Architecture MVC

  • Modèle (Battlefield, Area, Boat)
  • Vues (BattlefieldView, BattlefieldPanel)
  • Pattern Observateur via ModelListener
  • Communication via fireChange()

Joueurs et IA

  • HumanPlayer pour le joueur humain
  • RandomPlayer pour l'IA aléatoire
  • Interface Player commune
  • Gestion des tours et des attaques

Mécaniques de jeu

  • Placement des bateaux (put())
  • Attaques (assault())
  • Détection des coulés (sinkBoat())
  • Fin de partie (isOver())

Interface Graphique

  • Grilles interactives (AreaPanel)
  • Feedback visuel des tirs
  • Fenêtre principale (BattlefieldFrame)
  • Gestion des événements souris

Détails Techniques

Organisation en packages et classes clés :

  • Modèle : Battlefield, Area, Boat, Coord, Obus
  • Vues : BattlefieldView, BattlefieldPanel, AreaPanel
  • Contrôleurs : BattlefieldFrame, ClosedWindowListener
  • Joueurs : Player, HumanPlayer, RandomPlayer

Fonctionnalités algorithmiques principales :

  1. Validation du placement : isValid() dans Battlefield
  2. Gestion des tirs : assault() et mise à jour de l'état
  3. Détection fin de partie : isOver() vérifiant tous les bateaux
  4. Stratégie IA : Sélection aléatoire dans RandomPlayer

Bonnes pratiques de POO mises en œuvre :

  • Interface ModelListener pour le pattern Observateur
  • Interface Player pour les différents types de joueurs
  • Encapsulation stricte avec getters/setters
  • Classes spécialisées (Coord, Obus)
  • Séparation claire des responsabilités

Résultats Obtenus

Succès

  • Implémentation complète des règles du jeu
  • Interface graphique fonctionnelle avec Swing
  • Système de tour par tour opérationnel
  • IA basique mais fonctionnelle
  • Code bien structuré et documenté

Défis

  • Gestion des événements dans AreaPanel
  • Synchronisation modèle/vue via modelUpdated
  • Placement aléatoire des bateaux pour l'IA
  • Affichage correct des états de jeu

Conclusion

Ce projet de bataille navale a été une excellente opportunité pour mettre en pratique les concepts de programmation orientée objet et d'architecture MVC. L'implémentation des différentes vues (texte et graphique) a renforcé ma compréhension du pattern Observateur. La gestion des règles du jeu et de l'IA, bien que basique, a constitué un défi algorithmique intéressant. Ce projet démontre ma capacité à concevoir une application Java complète avec une interface utilisateur fonctionnelle.
