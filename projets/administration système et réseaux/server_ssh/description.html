Mise en Place et Sécurisation d’un Serveur SSH

SSH OpenSSH fail2ban

Description

Ce projet consiste à installer, configurer et sécuriser un serveur SSH sur une machine Linux pour des connexions distantes sécurisées.

SSH est un protocole essentiel pour l'administration système et réseau, permettant un accès chiffré aux serveurs.

Objectifs du projet

  • Installer et configurer un serveur SSH
  • Restreindre l’accès au serveur pour renforcer la sécurité
  • Mettre en place une authentification par clés SSH
  • Protéger le serveur contre les attaques bruteforce

Technologies et Outils Utilisés

  • Système d’exploitation : Linux (Ubuntu)
  • Protocole : SSH (Secure Shell)
  • Outils : OpenSSH, Fail2Ban, Firewall (UFW, iptables)
  • Sécurisation : Clés SSH, pare-feu, limitation des connexions

Étapes de réalisation

1. Installation du serveur SSH

Installation du serveur SSH et vérification du statut du service.

2. Configuration et sécurisation

Modification de la configuration SSH pour renforcer la sécurité.

3. Authentification par clés SSH

Génération d’une paire de clés et vérification de la connexion sans mot de passe.

4. Protection contre les attaques bruteforce

Installation de Fail2Ban et ajout de règles UFW pour restreindre l’accès.

Résultat et Perspectives

À l’issue de ce projet, le serveur SSH est sécurisé contre les attaques, et l’accès est limité aux utilisateurs autorisés avec authentification par clé.

Améliorations possibles :

  • Authentification multi-facteurs (MFA) pour SSH
  • Journalisation et alerte en cas de connexion suspecte
  • Mise en place d’un VPN pour masquer l’accès SSH

Disponible pour une alternance de 12/24 mois
